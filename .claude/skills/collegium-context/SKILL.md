# Collegium Context

Treat the project as an infrastructure of mobility governed by protocol.

## Core framing

- Do not reduce the project to an app de corridas, CRM, marketplace, backoffice, MLM, or generic bot automation.
- Treat the visible application as only one layer of a broader protocol, governance, and audit system.
- Preserve human final sovereignty for structural and sensitive decisions.
- Prefer extensibility, auditability, modularity, and traceability over local shortcuts.

## Required mental split

Separate the work into these layers:

- institutional
- protocol
- agentic
- application
- data and audit

Do not collapse those layers into a single implementation decision.

## Working rule

Before proposing or implementing anything, ask internally:

1. Is this institutional, protocol, or interface?
2. Does this affect agents, humans, or both?
3. Does this require an audit trail?
4. Does this affect reputation, economics, governance, or safety?
