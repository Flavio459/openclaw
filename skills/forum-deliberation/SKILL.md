# Forum Deliberation

Use this skill when the task belongs in `The Forum`, not in generic UI or raw execution.

## The Forum is for

- strategic discussion
- competing options
- risk framing
- incident analysis
- decision preparation
- evidence consolidation
- Chairman approval or rejection flow

## Required structure

When preparing or designing a `The Forum` interaction, organize the material as:

1. `topic`
2. `context`
3. `participants`
4. `evidence`
5. `options`
6. `risks`
7. `recommended path`
8. `chairman action`

## Routing rule

- If the task is about company interface only, route to `Cortex Command`
- If the task is about backstage engineering visibility, route to `Cortex Praetorium`
- If the task is about strategy, tradeoffs, and decision flow, keep it in `The Forum`

## Decision outputs

Prefer explicit actions:

- `approve`
- `reject`
- `defer`
- `escalate`

Do not leave strategic flows as vague chat transcripts.
